# 量化策略：不可外包的红线

> 从 SKILL.md 拆分。对量化策略，以下是 **cheap test 本身可信与否的最低条件**——cheap test 不可信，整个否决就不可信。必须在 skill 内过，不许只指向 playbook。

1. **泄漏 / 未来函数**：标签用 T 日信息→执行须 T+1 可得；滚动窗不含当日（`x[-253:-1]` 而非 `x[-252:]`）；无 NaN 分支偷偷回退 in-sample。
2. **最强基线对照**（非 strawman）：复杂方法必须先打过「不用核心新假设的最简单替代」。
3. **样本外定义清楚**：切分规则、有无重叠泄漏。
4. **噪声底 / 显著性**：要 CI/t 检验；「无法拒绝 IC=0」≠「证明 IC=0」。
5. **交易成本 / 容量**：换手×手续费进账；容量上限。
6. **多重比较校正**：第 N 次试验取 max 要声明（Deflated Sharpe / 至少标注试验次数）。
