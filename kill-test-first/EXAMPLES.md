# kill-test-first · 案例集（均为单点记录、非有效性证明）

> **案例 1 = 用法回放演示**（cheap test 关怎么用）；**案例 2 = skill 自身的审查史**（它自己怎么被多轮独立审查的）；**案例 3 = 先验对标关的回放演示**（门1 步骤2 怎么拦"重复造轮子"）。每个案例记录过程与诚实边界。
>
> ⚠ **证据等级声明**：案例 1 和案例 3 为对**已知结局**项目的事后回放，存在天然 hindsight。案例 2 为 skill 自身的审查过程记录。事后回放只能说明「该检查在这个回放里给出 STOP/REDESIGN 信号」，**不能证明当年必然拦下**。要真正证明 skill 有效，需前瞻测试或盲化回放（不知结局的 reviewer 用/不用 skill，比决策改变率 / 省时 / 误杀率）。

## 案例 1：某形态匹配择时策略的第0天假设（事后回放）

**背景**：某 DTW 形态匹配择时策略，在 **短时间内高速迭代十余个版本**，直到开发收尾后的复盘才发现核心假设理论上注定失败（滚动 IC 从早期版本就系统性为负，开发期间一次都没暴露）。⚠ 教训的要害不是「慢工出错」而是「**快工也救不了错方向**」——这期间每一版都在优化执行，却没有一版在问「这前提该不该做」。**越能快速迭代，越需要这道闸**，否则只是把错方向跑得更远。本案例把时钟拨回第0天，用 kill-test-first 门1跑一遍，看能否在开工前拦下。

### 门1 · 事前否决

**动作1 可证伪命题**
> 「在日频数据上，用 DTW 距离选出的 top-K 最相似历史形态，其后续 H 日收益对目标的后续 H 日收益有正的、样本外显著的预测力（RankIC>0，且优于不做形状匹配的朴素动量基线）。」

**动作2 最强反方（白板可得，先验）**
DTW 为「容忍时间弯折地匹配形状」而生，代价是牺牲精确时间位置；而预测未来最依赖的恰是最近一段位置精确的方向性动量。弯折会把「形状像但方向相反」的片段判成近邻。DTW 不创造信息，只换度量；弱有效市场下日频方向近鞅。
→ 先验给出的是「强红灯 + 一个可证伪机制」，不是终判，故需动作3。

**动作3 最便宜证伪实验（带朴素基线对照）**
某宽基指数 日线，过去 L=20 日形态 → DTW 取 top-K=20 历史近邻 → 用近邻后续 H=5 日收益预测目标后续5日收益，算样本外 RankIC。严格无未来泄漏（候选 s+H≤t-L）。**同时跑朴素动量基线 + 欧氏最近邻对照。** N=178。

| 方法 | 样本外 RankIC | 方向命中 |
|---|---|---|
| DTW 最近邻 | +0.0160 | 58.4% |
| 欧氏最近邻 | −0.0575 | 51.7% |
| 朴素动量（基线） | +0.0969 | 56.7% |

**动作4 死亡判据（跑前写死）**
`DTW RankIC≤0，或 DTW 不优于朴素动量基线 → STOP。`

### 裁决：STOP
DTW（+0.016）≈ 噪声零头（N=178 下一个标准误 ≈ 0.075，故 DTW 不显著；**与动量 +0.097 的差异是否显著本案例未检验**，勿当定论）。但在这个 cheap test 下，DTW **没给出继续建设的充分理由、且未打过朴素动量基线**——这就够触发 STOP。〔降级表述：不宣称「形状弯折注定是负价值」这种普遍结论，只说"在该检验下证据不支持继续"。〕这一枪很快就能打完、第 1 天就能亮红灯，却被推迟到十余个版本之后。

### 对抗式自检（hindsight 诚实）
- 动作2 反方 = DTW 定义级事实，纯先验，不作弊。
- 这一枪真跑、公开数据、严格无泄漏，数字非编造。
- 边界：测的是某宽基**指数**、特定参数（L20/H5/K20），**非原策略精确复现**；换标的/参数 +0.016 会浮动。故**不能证明「原策略必死」**——也不需要。事前否决只把举证责任压到建设之前：「凭什么有 alpha」在最便宜检验下答不上来、还输给朴素基线 → STOP。

### 本案例（回放演示）说明 / 改进了 skill
- ✅ 加载顺、门1四动作可照做、在这个回放里于第0天给出 STOP 信号（**不等于**证明当年必然拦下，见开头声明）。
- ⚖️ 诚实归因：本案例真正站得住的是「**复杂方法必须先打过朴素基线**」——这属于**通用研究脚手架**的价值，**不是 skill 独有 alpha**。skill 的价值应表述为「强制把通用基线 + 预登记前置并留审计痕迹」，而非「证明自己能发现失败方向」。
- 🔧 逼出改进：门1原只说「打一枪」，未强制带朴素基线——而「微弱正 IC 骗过有承诺的人」正是靠基线对照戳穿。已补入（对应 playbook §11.3）。

*脚本：某宽基指数日线；DTW 纯 numpy；无泄漏约束。事后回放。*

## 案例 2：kill-test-first 自己的多轮审查史

> 这个案例不是「用 skill 审别的项目」，而是「**skill 自己被独立审查的过程**」——一个关于"事前否决 + 独立审查"的 skill，自己完整走了一遍独立审查。
>
> ⚠ **证据边界**：以下是**单点过程记录**，**不是「多模型审查 > 单模型自审」的证明**。它只展示「在这一次里，独立审查抓到了自审没抓到的东西」——单点观测，不可外推为普遍结论。

### 审查链
| 阶段 | 角色 | 抓到 / 产出 | 卷入度 |
|---|---|---|---|
| 设计 | Claude | 轻量触发器初版 + 自审「对抗式自检」 | 设计者（非独立） |
| 回放 | Claude（真跑 cheap test） | 案例 1：DTW 未优于朴素动量基线 → 逼出"必带基线" | 自验（非独立） |
| 第1轮独立审 | **Codex** | 判「重大修改」：5 条硬伤（独立否决无机制、**死亡判据顺序 bug**、门1/门2 数据边界、过度声称、前置门槛外包）+ alpha 误定位 | 零卷入 |
| 整合大改 | Claude（整合官） | 全采纳 → 重写为完备协议版 | — |
| 第2轮独立审 | **Qwen** | 判「小改后定稿 · 0 高风险硬伤」：确认 Codex 5/5 解决；抓出大改**新引入**的问题（REDESIGN 可逃避 STOP、三态未覆盖象限）；"过度工程"上建议**砍**自身死亡判据 | 零卷入 |
| 调和 + 定稿 | 用户（decision owner） | 采纳 qwen 中风险小改；自身死亡判据**保留**（调和 Codex"加" vs qwen"砍"的对冲） | 决策人 |
| 后续增量复审 | ChatGPT | 门1步骤2先验对标关经零卷入增量复审，小改后保留 | 零卷入 |

### 这次记录到的几件事（诚实陈述，非普适结论）
- **自审有盲点**：Claude 自己的「对抗式自检」没发现「死亡判据在最小骨架里排到了结果之后」这个直接违背预登记的 bug——Codex 一眼抓到。
- **两轮独立审各抓各的、不重复**：Codex 抓"原版的硬伤"，qwen 抓"大改后新引入的问题"。没有一轮多余。
- **两个独立模型会对冲**：Codex 推"完备"、qwen 推"精简"，在「自身死亡判据」上方向相反——单个模型给不出这个对冲，最终由人调和。这是"多视角"的用法，不是缺陷。
- **收敛判据防住了无限诊断**：给 qwen 预设「无高风险硬伤即定稿、不许凑小改」，第2轮判 0 硬伤 → 收口，没滑向 playbook §9 警告的"无限诊断"。

### 留档（可追溯）
审查记录与提示词双件均位于项目归档目录。

### 它仍然没回答的（最重要的诚实）
**多轮审查定稿 ≠ 有效**。这个 skill 真正有没有 alpha，两轮纸面复审都答不了——只能靠前瞻 / 盲化使用验证（Codex 存在性拷问 + skill 自己的死亡判据①）。多轮审查定稿只说明"它内部自洽、无已知硬伤"，**不说明"它真能改变决策"**。那个问题，等真实使用。

## 案例 3：某项目零对标 FMTI 的事前对标回放

> **案例 3 = 门1 步骤2「先验对标」的回放演示**——和案例 1（cheap test 关）对称，演示**若第0天检索命中先例、会怎样给出 REDESIGN 信号**。
>
> ⚠ **证据等级声明**：对**已知结局**（已发现重合）的事后回放，存在 hindsight。只能说明「对标关在这个回放里给出 REDESIGN 信号」，**不能证明该项目第0天必然会搜到 FMTI**（取决于检索质量 / 术语找没找对）。

**背景**：某 LLM 报告完整性评估项目由 autonomous agent 自主设计十余类、数十项指标评 LLM 报告披露完整性，**全程零引用、零对标**。直到后期方法论评估，才发现它与 Stanford **FMTI**（Foundation Model Transparency Index，100 指标，已评多家主流厂商）高度重合。⚠ 教训不是"做错"，是"**做了个别人已做过、还更成熟的东西却不自知**"——agent 参数里其实"知道"FMTI，但在构建模式下不会主动调出来对标。把时钟拨回该项目第0天，用门1步骤2跑一遍。

### 门1 · 步骤2 先验对标

- **先要术语**：该项目想做的 = "给 LLM 厂商技术报告的披露/透明度打分排名" → 标准黑话 = *transparency index / disclosure evaluation / transparency reporting*。（不知道这串词就搜不到 → 这正是非专家 agent 的盲区。）
- **对抗式检索**（"假设我在重复造轮子，最可能重复了谁"）→ **Stanford FMTI**（Bommasani et al., 2023, arXiv:2310.12941）。
- **三问**：
  - ① 已有什么 = FMTI：100 指标 × 3 域（upstream/model/downstream）、多评分者、公开协议、已评主流厂商。
  - ② 我的差异 = 该项目有 0–3 质量分级 + 若干对抗式诚信类目 + 单报告深读；FMTI 是二元指标 / 厂商级 / 横截面。**差异真实但局部**。
  - ③ 重叠主体（披露完整性那部分）说不清"为什么不直接用 FMTI"。

### 裁决：REDESIGN
重叠主体大、且重叠部分说不清差异 → **不该从零造一套独立评估框架**，应改成「**FMTI 的对抗式补充模块**」（专做 FMTI 没有的"披露质量与评测诚信"那几关）。注意是 REDESIGN 不是 STOP——该项目的对抗式类目是真增量，杀掉可惜，但它不该以"独立框架"的身份存在。

### 诚实边界 / 归因（hindsight）
- 回放有 hindsight：结局已知才"搜得这么准"；第0天能不能搜到，取决于术语找没找对、用了几个来源。
- **诚实归因**：对标的价值是"**通用研究脚手架**"（任何研究都该先做 related work），**不是本 skill 独有 alpha**——和案例 1"必带基线属通用脚手架"同理。skill 的价值是「**强制前置 + 留审计痕迹**」，不是"能发现重复"。
- 单点回放，**非"对标关有效"的证明**；真有效需前瞻 / 盲化（不知结局的人用/不用这关，比重复率 / 省的沉没成本）。
- ⚠ **反例边界**：这个 REDESIGN 演示之所以顺，是因为已知 FMTI 后术语被精准回填。若第0天只搜「LLM 报告完整性 checklist」而没搜「transparency index」，本关**仍会漏掉 FMTI**——这正是为什么要术语三角验证（换 ≥3 个角度的词）。

### 这个回放如何收紧了 skill
- ✅ **支持（非坐实）**门1步骤2的**位置**：在这个回放里，对标比 cheap test 更早给出信号——该项目若先对标，不用跑评分实验就该 REDESIGN。但单点回放只是"支持/提示"，不能证明这个位置在一般情况下都成立。
- 🔧 逼出"**先要术语**"子步骤：该项目的根因是 agent 不知道这事叫 transparency index → 检索第一步必须是"问黑话"，否则非专家 + agent 双盲。
