# Step 5：多后端验证（重要项目）

> 从 SKILL.md 拆分。对复杂或高风险项目的 CLAUDE.md 进行跨模型交叉验证。

## 触发条件（满足任一）

- 项目复杂度高（≥3 个子系统/模块）
- CLAUDE.md 改动幅度 >30%
- 修改了核心停止条件或架构约束

## 执行步骤

- `[ ]` 将 CLAUDE.md 草案 + 评估维度写成一份审查 prompt（模板见 `_reviews/prompts/` 下的既有审查 prompt）
- `[ ]` **同一份 prompt** 并行投给 ≥2 个不同后端（不可同一后端换壳）
- `[ ]` 结果分类：**收敛**（≥2/3 一致）→ 采纳 / **互补**（仅 1/3 发现）→ 评估后采纳 / **冲突**（A 说砍 B 说留）→ 提交用户裁决
- `[ ]` 审查报告 + 最终 CLAUDE.md 一并归档至 `_reviews/`

> **安全约束**：审查 prompt 不得包含仓库机密（API key/内部路径/私有数据）。使用脱敏摘要，敏感信息用占位符替代。
