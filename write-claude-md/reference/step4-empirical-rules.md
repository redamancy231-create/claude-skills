# Step 4：三项实证规则

> 从 SKILL.md 拆分。经过实际项目验证的硬规则，不可跳过。

- `[ ]` **a. 白名单指针化**：模型名/工具版本/可用后端清单不放 CLAUDE.md → 放 `project_status.md` 或 `reference_files.md`，CLAUDE.md 只写"见 X 文件"
- `[ ]` **b. 术语显式定义**：项目中非默认含义的术语必须写清楚定义。判据——如果术语的默认含义会让 Agent 做错误操作，就必须定义（如"试跑"在方法论项目=完整 AI 协作周期，不是脚本执行）
- `[ ]` **c. 不维护"最后更新日期"**：CLAUDE.md 只在操作指令变更时才修改，不因其他文件变化而机械更新。变更历史由 git log 追踪
