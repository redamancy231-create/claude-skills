# Session End 检查清单

> 从 SKILL.md 拆分。按顺序执行，逐条打勾。

## 2.1 项目状态

- [ ] **更新 project_status.md**，保持以下固定四段结构（顺序不可变，段不可删）：
  1. `## 项目状态: [项目名]`
  2. `- 当前阶段: [阶段名]` — 如未变则写"同前"
  3. `- 本轮完成: [≤5条，过去式动词]` — 如"修复了X/创建了Y/更新了Z"
  4. `- 发现的问题: [问题简述或"无"]`
  如文件不存在，用上述模板创建；已存在则按固定四段替换对应内容。

- [ ] **更新项目记忆文件**：`project_status.md` 更新后，检查 `~/.claude/projects/<project-hash>/memory/` 下本项目对应的记忆文件（通常文件名含项目关键词，如 `project_<项目名>.md`），逐条核对其中的事实断言/待办行/状态声明是否与刚更新的 `project_status.md` 一致。**日期新鲜≠内容准确**——2 天前的待办行可能在本次会话中已清掉。发现过期内容立即更新。判断方法：打开记忆文件，找"待办"/"待完成"/"剩余"等关键词，与 `project_status.md` 的 Next Steps 交叉比对。

## 2.2 发现沉淀

- [ ] **新发现→Retrospect**：方法论/流程/工具层面的发现 → 写 Retrospect（项目 `_reviews/` 目录下，命名 `retrospect_YYYY-MM-DD.md`）或标"Retrospect 候选"（附一句话理由）。**判断是否值得写的 3 个问句**：(a) 有什么发现是下一会话不该不知道的？ (b) 本次犯了什么错下次可以避免？ (c) 有什么跨项目通用的？——命中任一条即写，全不命中则标"Retrospect 候选"并附一句话理由。
- [ ] **跨任务通用教训→memory/**：适用于其他项目的教训写 memory 文件到 `~/.claude/projects/<project-hash>/memory/`（最少 frontmatter 字段：`name`/`description`/`metadata.type`；body + 关联链接）；仅限本项目的记在 project_status.md
  - **判定标准**："如果另一个不相关的项目可能直接受益于此教训 → 跨项目通用，写 memory/；如果教训仅在当前项目的特定上下文中有意义 → 本项目，记 project_status.md"

## 2.3 未完成任务

- [ ] **写标准化 Next Steps**（如有未完成任务）：

```
操作动词 + 具体文件/命令(绝对路径) + 优先级(P0/P1/P2) + 依赖(等什么完成后才能做)
```

例：
```
运行 projects/my-project/test.py 验证结果模板 → P0 → 等数据更新后
更新 strategy.py 因子权重 → P1 → 等V3.7回测完成后
```

- **P0**：阻塞下一阶段/有时限/不进下一步项目废
- **P1**：本阶段应完成但不阻塞下一阶段启动
- **P2**：锦上添花/可被后续会话自主判断

## 2.4 文件索引

- [ ] **更新 reference_files.md**：采用以下最小模板确保机器可解析：
  ```
  # 关键文件索引

  > 最后更新: YYYY-MM-DD

  ## 代码
  - [文件名](相对路径) — ≤15字用途

  ## 数据
  - [文件名](相对路径) — ≤15字用途

  ## 文档
  - [文件名](相对路径) — ≤15字用途
  ```
  - 如不存在：用上述模板创建（代码/数据/文档三区至少各占一行，无该类型则写"无"）
  - 已存在：保持现有结构，在对应分区末尾追加新增项（格式 `- [文件名](路径) — 用途`），对已删除文件在行末标 `(已删除 YYYY-MM-DD)` 而非直接移除行
  - 不含临时文件/中间产物；关键文件=主代码/主数据/主文档/配置文件
